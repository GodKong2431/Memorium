# Gemini Visual Prompt

아래 프롬프트를 그대로 Gemini나 GPT에 전달해서 사용합니다.

```text
다음 문서는 Unity 프로젝트 발표자료를 만들기 위한 시각 중심 슬라이드 기획서다.

중요 조건:
- 슬라이드 본문에 파일명, 클래스명, 메서드명, 스크립트 경로를 쓰지 마라.
- 발표자료가 개발 문서처럼 보이지 않게 작성하라.
- 현재 프로젝트에 실제로 없는 기능, 화면, 연출을 추측해서 추가하지 마라.
- 제공된 문서에 적힌 범위 안에서만 설명하라.
- 모든 슬라이드에 Before/After를 억지로 넣지 마라.
- Before/After는 전후 차이가 직관적인 기술에만 사용하라.
- 어떤 슬라이드는 실제 화면 + 플로우차트가 더 적합하고, 어떤 슬라이드는 구조도나 절차 비교가 더 적합하다.
- 각 슬라이드마다 가장 설득력 있는 시각 자료 방식을 선택하라.
- 코드를 설명하는 방식보다 문제, 해결 방식, 효과, 사용자 경험, 운영 개선을 중심으로 작성하라.
- 현재 프로젝트에 실제로 들어갈 만한 화면 예시와 이미지 구성을 구체적으로 제안하라.
- 특히 스킬 슬라이드는 프리셋 전환보다 `스킬의 기본 정보와 모듈 1~5 구조`, `확장 슬롯 상태`, `모듈 정보가 UI와 전투 데이터에 연결되는 흐름`을 중심으로 다뤄라.

출력 형식:
slide 1
- 제목
- 핵심 메시지
- 슬라이드 본문 bullet 3개
- 가장 적합한 시각 구성
- 추천 레이아웃
- 플로우차트 또는 비교 요소
- 현재 프로젝트에서 넣기 좋은 실제 화면
- 발표자 멘트

slide 2
...

목표:
- 아래 8개 기술 주제를 각각 1장 발표자료로 만든다.
- 면접 발표 또는 포트폴리오 발표에 바로 쓸 수 있는 수준으로 구성한다.
- 실제 팀 프로젝트에서 기술 문제를 구조적으로 해결한 사례처럼 보이게 한다.

디자인 방향:
- 배경은 밝고 깔끔한 톤
- 포인트 색은 네이비, 청록, 오렌지 계열 중 2색 조합
- 텍스트보다 시각 자료 비중이 높아야 한다
- 슬라이드마다 한 가지 메시지만 남기고 복잡한 설명은 줄인다

추가 조건:
- 각 슬라이드는 문제 -> 해결 -> 효과 흐름이 읽히게 작성
- 본문은 짧고 강한 문장으로 작성
- 발표자가 실제로 말하기 쉬운 문장으로 정리
- Before/After가 비효율적인 슬라이드에는 구조도, 단계 비교, GIF, 실제 UI 화면 중심으로 구성
- 실제 프로젝트에서 촬영하거나 캡처할 만한 장면을 구체적으로 제안

아래 문서를 기반으로 슬라이드를 작성하라:
[여기에 GEMINI_VISUAL_SLIDE_PACK.md 내용 붙여넣기]
```

---

## 사용 팁

- 결과가 길어지면 `1~4장`, `5~8장`으로 나눠 요청하는 편이 좋습니다.
- 실제 스크린샷이나 GIF가 있으면 프롬프트 뒤에 함께 첨부해서 해당 슬라이드에 우선 배치하도록 지시하면 결과 품질이 좋아집니다.
- 발표자 멘트까지 함께 받고 싶으면 마지막에 `각 슬라이드의 발표자 멘트는 실제 면접 발표 톤으로 2~3문장으로 작성`을 추가하면 됩니다.
